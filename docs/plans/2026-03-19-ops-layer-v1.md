# Ops Layer V1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a first-class operations layer that makes session workflows explicit, testable, and recoverable, while preserving current multi-agent and worktree behavior.

**Architecture:** Introduce an `ops` domain with explicit operation models, a command runner abstraction, and focused services for worktree lifecycle and git polling. Refactor existing runtime logic out of `AppState`/`AgentDriverManager` into orchestrated operations with typed outcomes. Keep UI contracts stable, but route critical flows through `OpsCoordinator`.

**Tech Stack:** Flutter/Dart, Provider, existing `Process.run/Process.start`, `flutter_test`

---

## Module Boundaries and Interfaces (V1)

### Boundary A: Operation Domain (new)
- Path: `lib/ops/`
- Responsibility: define operation identity, status, attempts, timestamps, and typed results.
- Public interfaces:
  - `OperationRecord` (`id`, `kind`, `sessionId`, `status`, `attempt`, `startedAt`, `endedAt`, `error`)
  - `OperationResult<T>` (`success(value)` / `failure(error)`)
  - `OpsLogger` (`started`, `completed`, `failed`)

### Boundary B: Infrastructure Adapters (new)
- Path: `lib/services/ops/`
- Responsibility: isolate shell/process/git execution and polling details.
- Public interfaces:
  - `CommandRunner.run(...)`
  - `WorktreeOps.create(...)`, `WorktreeOps.remove(...)`
  - `GitWatcher.watch(...)`, `GitWatcher.refreshOnce(...)`

### Boundary C: Orchestration (new)
- Path: `lib/ops/ops_coordinator.dart`
- Responsibility: execute session-level workflows (create session, send prompt, refresh git), emit operation lifecycle events, centralize retry/idempotency policy.
- Public interfaces:
  - `createSessionFlow(...)`
  - `sendMessageFlow(...)`
  - `closeSessionFlow(...)`

### First 3 Refactors in this plan
1. Refactor shell/git calls behind `CommandRunner` + `WorktreeOps`.
2. Refactor `AgentDriverManager.createDriver` to use `WorktreeOps` and typed failures.
3. Refactor `AppState` git timer logic to `GitWatcher` service.

```dart
// lib/ops/models/operation_result.dart
class OperationResult<T> {
  final T? value;
  final String? error;
  bool get isSuccess => error == null;

  const OperationResult._({this.value, this.error});
  factory OperationResult.success(T value) => OperationResult._(value: value);
  factory OperationResult.failure(String error) =>
      OperationResult._(error: error);
}
```

```dart
// lib/ops/models/operation_record.dart
enum OperationKind { createSession, sendMessage, refreshGit, closeSession }
enum OperationStatus { running, completed, failed }

class OperationRecord {
  final String id;
  final OperationKind kind;
  final String sessionId;
  final OperationStatus status;
  final int attempt;
  final DateTime startedAt;
  final DateTime? endedAt;
  final String? error;

  const OperationRecord({
    required this.id,
    required this.kind,
    required this.sessionId,
    required this.status,
    required this.attempt,
    required this.startedAt,
    this.endedAt,
    this.error,
  });

  factory OperationRecord.started({
    required OperationKind kind,
    required String sessionId,
    int attempt = 1,
  }) {
    final now = DateTime.now();
    return OperationRecord(
      id: '${kind.name}-${now.microsecondsSinceEpoch}',
      kind: kind,
      sessionId: sessionId,
      status: OperationStatus.running,
      attempt: attempt,
      startedAt: now,
    );
  }

  OperationRecord completed() => OperationRecord(
        id: id,
        kind: kind,
        sessionId: sessionId,
        status: OperationStatus.completed,
        attempt: attempt,
        startedAt: startedAt,
        endedAt: DateTime.now(),
      );

  OperationRecord failed(String reason) => OperationRecord(
        id: id,
        kind: kind,
        sessionId: sessionId,
        status: OperationStatus.failed,
        attempt: attempt,
        startedAt: startedAt,
        endedAt: DateTime.now(),
        error: reason,
      );
}
```

```dart
// lib/services/ops/command_runner.dart
import 'dart:io';

class CommandOutcome {
  final int exitCode;
  final String stdout;
  final String stderr;
  const CommandOutcome(this.exitCode, this.stdout, this.stderr);
  bool get ok => exitCode == 0;
}

class CommandRunner {
  Future<CommandOutcome> run(
    String executable,
    List<String> args, {
    String? workingDirectory,
  }) async {
    final result = await Process.run(
      executable,
      args,
      workingDirectory: workingDirectory,
    );
    return CommandOutcome(
      result.exitCode,
      result.stdout.toString(),
      result.stderr.toString(),
    );
  }
}
```

```dart
// lib/services/ops/worktree_ops.dart
import '../../errors/errors.dart';
import 'command_runner.dart';

class WorktreeOps {
  WorktreeOps(this._runner);
  final CommandRunner _runner;

  static List<String> buildCreateArgs({
    required String branchName,
    required String worktreePath,
    required String parentBranch,
  }) {
    return ['worktree', 'add', '-b', branchName, worktreePath, parentBranch];
  }

  Future<void> create({
    required String repoPath,
    required String branchName,
    required String worktreePath,
    required String parentBranch,
  }) async {
    final out = await _runner.run(
      'git',
      buildCreateArgs(
        branchName: branchName,
        worktreePath: worktreePath,
        parentBranch: parentBranch,
      ),
      workingDirectory: repoPath,
    );
    if (!out.ok) {
      throw AppError(
        ErrorCode.worktreeCreationFailed,
        message: 'Failed to create worktree',
        details: out.stderr,
      );
    }
  }

  Future<void> remove({
    required String repoPath,
    required String worktreePath,
  }) async {
    await _runner.run(
      'git',
      ['worktree', 'remove', worktreePath, '--force'],
      workingDirectory: repoPath,
    );
  }
}
```

```dart
// lib/services/ops/git_watcher.dart
import 'dart:async';

typedef FetchSnapshot<T> = Future<T> Function();

class GitWatcher<T> {
  GitWatcher({required FetchSnapshot<T> fetch}) : _fetch = fetch;
  final FetchSnapshot<T> _fetch;

  Stream<T> watch({Duration interval = const Duration(seconds: 3)}) async* {
    while (true) {
      yield await _fetch();
      await Future<void>.delayed(interval);
    }
  }

  Future<T> refreshOnce() => _fetch();
}
```

```dart
// lib/ops/ops_coordinator.dart
import 'models/operation_record.dart';
import 'models/operation_result.dart';
import 'ops_logger.dart';

class OpsCoordinator {
  OpsCoordinator({required this.logger});
  final OpsLogger logger;

  Future<OperationResult<void>> createSessionFlow() async {
    final op = OperationRecord.started(
      kind: OperationKind.createSession,
      sessionId: 'pending',
    );
    logger.started(op);
    try {
      final done = op.completed();
      logger.completed(done);
      return OperationResult.success(null);
    } catch (e) {
      final failed = op.failed(e.toString());
      logger.failed(failed);
      return OperationResult.failure(e.toString());
    }
  }

  Future<OperationResult<void>> sendMessageFlow() async {
    return OperationResult.success(null);
  }
}
```

```dart
// lib/ops/in_memory_ops_logger.dart
import 'ops_logger.dart';
import 'models/operation_record.dart';

class InMemoryOpsLogger implements OpsLogger {
  InMemoryOpsLogger({this.maxEntries = 200});
  final int maxEntries;
  final List<OperationRecord> _entries = [];
  List<OperationRecord> get entries => List.unmodifiable(_entries);

  @override
  void started(OperationRecord operation) => _push(operation);

  @override
  void completed(OperationRecord operation) => _push(operation);

  @override
  void failed(OperationRecord operation) => _push(operation);

  void _push(OperationRecord operation) {
    _entries.add(operation);
    if (_entries.length > maxEntries) {
      _entries.removeRange(0, _entries.length - maxEntries);
    }
  }
}
```
